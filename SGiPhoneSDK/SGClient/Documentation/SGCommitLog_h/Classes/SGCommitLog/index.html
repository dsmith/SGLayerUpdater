SGCommitLog

A small implementation of a commit log that contains replay functionality.

Superclass: NSObject
Declared In: SGCommitLog.h

Discussion

The directory format is stored as:

#{commit_log_name}/#{username}/#{key}/#{commit}-#{current_timestamp}

Each commit is stored within the key of a username. Commits are stored in a file called commit that has a timestamp appended to it.

Discussion

Commits for the user are loaded into memory and then sent of to the delegate for some replay action. Once the commits have been replayed, they are deleted from disk.
